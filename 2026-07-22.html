💡 CLAUDE CODE TIP OF THE DAY

Since v2.1.215, Claude Code no longer auto-runs /code-review or /verify on its own — you have to invoke them explicitly.

Get the most out of this: add a "done criteria" block to your CLAUDE.md (e.g. "a feature is done when: all tests pass, no console errors, mobile viewport tested, no hardcoded strings"). When you run /code-review, Claude checks your actual list rather than guessing what "done" means. You can also run /verify after a specific change to spot regressions before pushing.

This turns code review from a vibe check into a verifiable checklist — especially useful when you're building Omniuss client tools and handing off to non-technical operators.
