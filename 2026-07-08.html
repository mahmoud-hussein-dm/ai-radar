⚡ CLAUDE CODE TIP — VERIFIABLE DONE-CRITERIA

Write explicit done-criteria, not vague instructions.

Instead of: "build me a product description generator"

Write: "Build a product description generator. Done when: (1) running node generate.js 'black t-shirt M' outputs 100 words, (2) missing input handled gracefully with a clear error, (3) npm test passes."

Claude self-checks against explicit criteria. Vague tasks produce vague results. This one habit cuts your revision cycles in half.
