CLAUDE CODE TIP OF THE DAY

Tip: Write verifiable done-criteria in CLAUDE.md

Most CLAUDE.md files tell Claude what the project is. The unlock is telling Claude how to know it's *done*.

Add a section like:

## Done criteria
A task is complete only when:
- [ ] The change runs without errors locally
- [ ] The relevant test passes (run: npm test -- --grep <task>)
- [ ] The output matches the example in /examples/

Claude will self-check against these before wrapping up — no more "I think this is done" with a broken build.

For ecom automations: your done criteria might be "Klaviyo flow triggered for test@email.com" or "Shopify order #1234 status updated to fulfilled". Make it checkable, not vague.
