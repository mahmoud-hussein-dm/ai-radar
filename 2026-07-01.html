CLAUDE CODE TIP OF THE DAY

Use verifiable done-criteria. End every task you give Claude Code with a "Done when:" line that describes exactly what success looks like — e.g. "Done when: the /api/products endpoint returns a 200 with all RABBIO variants and `npm test` passes with 0 failures." This gives Claude a concrete exit condition to check against, not just a feeling that the code "looks done." It catches regressions before you review and makes complex tasks much more reliable.
